# Requisitos Funcionais

> Apenas requisitos com status **Aceito** estão listados neste documento.

| ID | Título | Descrição | Status |
|----|--------|-----------|--------|
| RF-01 | RF-03: Previsão de Resultados Processuais | O sistema deve prever os possíveis resultados de um processo legal. Isso inclui extinção, improcedência, procedência parcial, procedência e acordo. | Aceito |
| RF-02 | RF-04: Geração de Proposta de Acordo | O sistema deve, em caso de recomendação de acordo, sugerir um valor de indenização. Esta proposta deve ser baseada em dados históricos e tendências. | Aceito |
| RF-03 | RF-05: Otimização da Política de Acordos | O sistema deve gerar e otimizar uma política de acordos do banco. Esta política será um conjunto de regras (árvore de decisão) baseada em dados. | Aceito |
| RF-04 | RF-01: Análise de Casos Legais | O sistema deve receber como entrada os dados de um caso legal específico para análise. Isso inclui informações relevantes para a tomada de decisão entre acordo e processo. | Aceito |
| RF-05 | RF-02: Recomendação de Acordo ou Processo | O sistema deve gerar uma recomendação clara entre acordo ou processo para cada caso. A recomendação deve ser acompanhada de uma probabilidade de sucesso. | Aceito |
| RF-06 | RF-07: Visualização de Dados e Resultados | O sistema deve apresentar um dashboard intuitivo para visualização das recomendações e métricas de desempenho. Isso inclui a performance da política de acordos. | Aceito |
| RF-07 | RF-08: Adaptação a Diferentes Processos | O sistema deve ser adaptável para analisar e gerar políticas para diferentes tipos de processos legais. Isso garante a flexibilidade da solução. | Aceito |
| RF-08 | RF-09: Análise de Jurisprudência Histórica | O sistema deve incorporar dados históricos e jurisprudência para embasar suas análises e recomendações. Isso inclui variações por estado e comarca. | Aceito |
| RF-09 | RF-10: Captura de Feedback do Usuário | O sistema deve permitir que o advogado forneça feedback sobre as recomendações geradas. Este feedback será utilizado para o refinamento contínuo do modelo. | Aceito |
| RF-10 | RF-11: Simulação de Políticas Dinâmicas | O sistema deve simular o impacto de diferentes políticas de acordo antes de sua aplicação real. Isso permite a avaliação de cenários e otimização. | Aceito |
| RF-11 | RF-12: Justificativa da Recomendação | O sistema deve fornecer uma justificativa clara e compreensível para cada recomendação de acordo ou processo. Isso ajuda o advogado na tomada de decisão. | Aceito |
